C++ Assignment: Simple Calculator

Write a short C++ program that displays a menu of arithmetic operations and performs the selected calculation using a switch-case selection structure. Keep the entire solution in main() and follow the header and style rules from earlier labs.

Program Requirements

  • Include the standard header block at the top of your .cpp file (same format used in earlier labs).
  • All logic must be contained in main(). Do not write any additional functions or methods.
  • Menu & Selection: Display a simple menu and let the user choose an operation using a single character: '+', '-', '*', or '/'.
  • Switch–case: Use a switch statement on the chosen character to select which arithmetic operation to perform.
  • Operands: In the chosen operation's code block, prompt the user to enter two numeric values as doubles, perform the calculation, and display the result.
  • Loop (optional): You may wrap the menu in a loop to allow multiple calculations, or run the menu once and end the program.
  • Error handling: Provide a message for an unknown operator; if you implement the loop, let the user exit cleanly.
  • Division: If division is selected, include a basic check to avoid dividing by zero (e.g., ask for a non‑zero second value or show an error message).
Input format: Read a single character for the operator from the menu (+ - * /). Then read two double values as operands in the selected case. If you implement a loop, re-prompt for the operator each time; allow the user to quit by entering any non-operator key (or by a separate prompt).

Output Requirements

  • Show the chosen operation and the computed result in a clear, readable format.
  • For division by zero, show an explanatory message instead of a numeric result.
  • For an unrecognized operator, show an appropriate error message.

Before You Submit

  • Header block is complete and correctly formatted.
  • All code is inside main(); no additional functions or methods are defined.
  • Program compiles and runs without errors.
  • Menu accepts + - * / and performs the correct arithmetic with two double operands.
  • Division by zero is handled gracefully.
  • (If loop implemented) The program repeats and exits cleanly when requested.

Instructor notes for future changes

  • Specify exact prompts you expect (e.g., the operator prompt and operand prompts) to standardize I/O across submissions.
  • Define exit behavior if using a loop (e.g., "enter q to quit" or "any non‑operator key exits").
  • State the desired numeric formatting for results (e.g., show fixed notation with two decimal places, or default formatting).
  • Clarify whitespace rules (whether spaces around operators are required in output, and how to display the full expression, e.g., 3.0 + 4.5 = 7.5).
  • Confirm input assumptions (e.g., whether to accept scientific notation, and whether to reprompt on invalid numeric input for operands).
