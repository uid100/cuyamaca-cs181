Standard C++ Program Header & Submission Guide

Course: CS 181 · Instructor: D. Sullivan

Purpose

This page defines the required header comment that must appear at the top of every C++ source file you submit in CS 181. It also includes the compiler/build instructions and submission expectations. Consistent headers support clarity, grading efficiency, and professional documentation habits.

Lines in your code, including the header, should be no longer than 80 characters, but never > 100 characters to ensure readability across different editors and devices. Use line breaks and indentation to maintain this limit while keeping the header organized and easy to read.

Required C++ Header (Copy & Fill In)
/*
  File:        <filename.cpp>
  Author:      <Your Name>
  Course:      Cuyamaca College - CS 181
  Instructor:  D. Sullivan
  Date:        <Submission Date (e.g., 2026-02-03, formatted YYYY-MM-DD)>

  Description:
      <A brief summary (2–4 sentences) describing what the program does.>

  Inputs:
      <List any user inputs or "None" if there are none.>

  Outputs:
      <List the output the program produces.>

  Notes:
      <Optional: assumptions, limitations, citations, acknowledgments.>
  */
Example (Filled-In Header)
/*
  File:        circle_area.cpp

  Description:

Compiler / Build Instructions

Use a recent GNU g++ or compatible compiler targeting at least C++17. On most systems, the following commands will compile and run a single-file program:

Linux / macOS (Terminal)
# Compile
  g++ -std=c++17 filename.cpp -o program

  # Run
  ./program
Windows (PowerShell / CMD with MinGW-w64)
:: Compile
  g++ -std=c++17 filename.cpp -o program.exe

  :: Run
  program.exe
Multi-File Projects (Optional)

If your assignment uses multiple source files, compile them together:

g++ -std=c++17 main.cpp helper.cpp -o program
Common Flags (Optional)
  • -Wall -Wextra -Wpedantic: enable useful warnings
  • -O2: optimize for speed (optional)
  • -g: include debug symbols for use with gdb
Submission Requirements
  • Each .cpp file must include the required header at the top.
  • Code must compile using g++ -std=c++17 without errors.
  • Provide all source files needed to compile and run.
  • Follow assignment-specific naming (e.g., sum_integers.cpp, circle_area.cpp).
  • Submit via the designated LMS dropbox before the deadline.
Academic Integrity & Notes
  • Write your own code. If you consult resources, cite them in the header’s Notes section.
  • Do not share code or accept code from others.
  • Keep descriptions specific to your own implementation to demonstrate understanding.
  • If using an IDE (VS Code, CLion, etc.), ensure your project still builds with g++ as shown above.
