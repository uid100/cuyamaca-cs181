Arrays and Pointers

In C++, arrays and pointers are closely related. The name of an array acts like a pointer to its first element in memory. This allows pointer notation to be used to access and traverse array elements.

Accessing Array Elements with Pointers

Given an array, the array name represents the address of the first element. You can store this address in a pointer. 


int numbers[4] = {10, 20, 30, 40};
int* ptr = numbers;   // points to numbers[0]

Dereferencing the pointer accesses the value at that address: 


std::cout << *ptr;       // 10
std::cout << *(ptr + 1); // 20

Pointer arithmetic automatically accounts for the size of the data type.

Pointer Arithmetic and Arrays

Adding an integer to a pointer moves it forward by that many elements in the array—not bytes. 


int* ptr = numbers;

ptr = ptr + 2;          // now points to numbers[2]
std::cout << *ptr;      // 30

This works because the compiler knows the data type the pointer refers to.

Traversing an Array Using a Pointer

Pointers can be used to loop through an array by advancing the pointer from the first element to the last. 


int numbers[4] = {10, 20, 30, 40};
int* ptr = numbers;

for (int i = 0; i < 4; i++) {
    std::cout << *ptr << " ";
    ptr++;
}

Each increment moves the pointer to the next array element.

Key Takeaways

  • An array name acts like a pointer to the first element
  • Pointers can access array elements using dereferencing
  • Pointer arithmetic moves by elements, not bytes
  • Pointers provide an efficient way to traverse arrays
