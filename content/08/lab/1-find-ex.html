Programming Exercise: Linear Search in an Integer Array

In this exercise, you will write a simple C++ program that demonstrates a linear search. A linear search checks each element in an array one at a time until a target value is found or the end of the array is reached. To practice good program design, the search logic must be placed in its own function.

Learning Objectives

  • Practice working with integer arrays
  • Write and call user-defined functions
  • Use loops to process array data
  • Understand how a linear search works
  • Relate program behavior to O(n) time complexity

Program Requirements

  • Declare and initialize an integer array with at least 10 values. This array should be declared as a constant near the top of your source file, outside of any function, to allow for easy modification.
  • Declare a constant int NOT_FOUND = -1;
  • Prompt the user to enter an integer to search for
  • Implement the linear search in a separate function
  • Call the search function from main()
  • If the value is found, return the index and display its value (from main(), not in the search function)
  • If the value is not found, return NOT_FOUND

Search Function Prototype

Your linear search function should use the following prototype: 


int linearSearch(const int arr[], int size, int target);

This function should return the index of the target value if it is found, or NOT_FOUND if the value is not in the array.

Instructions

  1. Create a new C++ source file.
  2. Define and initialize an integer array in main().
  3. Prompt the user for a number to search for.
  4. Call the linearSearch function, passing the array, its size, and the search value.
  5. Use the return value to determine what message to display.

Hints

  • Arrays are passed by reference, so no copying occurs.
  • Use a for loop inside the search function.
  • Remember that array indexes start at 0.

Sample Output

Enter a number to search for: 42
Value found at index 5.
  
Enter a number to search for: 99
Value not found in the array.

When finished, you should understand why this approach checks each element sequentially and why linear search is considered an O(n) algorithm.

You should also understand why the size of the array is a required parameter for the search function, since arrays do not carry size information with them.

Except for the test data in the array, there should be no hard-coded values in your program.

Like all programming exercises, don't forget the required header content at the top of your source file, including comments that describe the program's purpose, author, and date.
