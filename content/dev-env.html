Development Environment

This course focuses on fundamental C++ programming using standard, crossâ€‘platform language features. To ensure consistency across Windows, macOS, and Linux systems, all assignments must follow the environment policy below:

  • Students may use any code editor they prefer, including VS Code, CLion, Xcode, or Vim.
  • Only a standard-compliant compiler may be used:
    • Windows: MinGW-w64 or WSL g++
    • macOS: clang++ (included with Xcode tools)
    • Linux: g++
  • Do NOT use Visual Studio project templates (e.g., .sln, .vcxproj) or precompiled headers such as pch.h or stdafx.h.
  • All submitted work must compile using a standard command such as: 
    g++ -std=c++17 -o program main.cpp
  • Submitted files must include only:
    • .cpp source files
    • .h or .hpp header files
    • Optional: a brief README.txt
  • Submit assignments as a .zip file containing your lab folder only.

This policy ensures that code remains portable, simple, and universally compilable on all platforms used in the course.
