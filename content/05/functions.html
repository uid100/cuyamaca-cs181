Introduction to Modular Code

Modular code means breaking a program into smaller, reusable, and well‑defined pieces. In C++, these pieces are usually functions. Each function handles one task, which helps make programs easier to read, test, and maintain.

Why modular code?

  • Improves readability by organizing code into meaningful parts
  • Makes debugging easier—problems are isolated in smaller sections
  • Allows reusability across multiple programs
  • Encourages team collaboration by dividing tasks into logical units

Understanding Return Types

Every C++ function has a return type, which tells the program what kind of value the function will give back when it finishes running.

Examples:

  • int – returns an integer
  • double – returns a decimal value
  • bool – returns true or false
  • void – returns nothing

The return type appears at the very beginning of a function declaration or definition.

Understanding Parameters

Parameters are values that a function needs in order to do its job. They act as variables inside the function and are supplied when the function is called.

Examples:

  • int add(int a, int b) – requires two integers
  • void printMessage(string msg) – takes a string to display

Parameters make functions flexible and reusable with different inputs.

Defining and Calling Functions

A function definition is where the function’s code actually lives. It includes the return type, the function name, parameters, and the statements that run. 

int add(int a, int b) {
    return a + b;
}

A function call is how you use the function elsewhere in your program. 

int result = add(3, 4);

The call supplies values (arguments) that match the function's parameters.

Using Function Prototypes

A function prototype is a declaration placed before main() that tells the compiler a function exists and what its return type and parameter types are.

Example prototype with parameter names:

int add(int a, int b);

Parameter names in prototypes are optional, but including them helps readability:

int add(int, int);     // also valid, less readable

Technically, prototypes can be optional if the function is defined before it is used. However, in this course we will always use prototypes for clarity and best practice.
