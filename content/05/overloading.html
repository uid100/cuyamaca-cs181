Function Overloading

In C++, function overloading allows multiple functions to have the same name as long as they have different parameter lists. This makes functions easier to read and use because related tasks can share a name.

Examples of overloading:

int add(int a, int b) {
    return a + b;
}

double add(double a, double b) {
    return a + b;
}

int add(int a, int b, int c) {
    return a + b + c;
}

The compiler chooses which version to call based on the number and types of the arguments. Overloading helps keep function names meaningful without inventing variations like addInts or addThree.

Caution: Overloading and Default Parameters

Default parameters can sometimes cause confusion when combined with overloaded functions. The problem happens when two overloaded functions could both match a call because of default values.

Example of a confusing situation:

void show(int x);
void show(int x, int y = 0);

If you call: 

show(5);

The compiler cannot tell which version to use:

  • show(int x) → exact match
  • show(int x, int y = 0) → also a match because y has a default

This creates an ambiguity error, which stops the program from compiling.

How to Avoid These Problems
  • Do not overload a function in a way that only differs by a default parameter.
  • Use default parameters in one version of a function, not several.
  • Make sure each overloaded version clearly requires different arguments.
  • When teaching beginners, keep overloading simple: vary the number or types of parameters, not default values.

Used carefully, overloading and default parameters are powerful tools, but mixing them without planning can lead to confusing or ambiguous code.
