Greeting Function with Default Parameter

In this lab, you will write and call a custom C++ function named greet(). The function must accept a single std::string parameter passed by value, and it must include a default argument value.

All work must be completed in a single file named:

Greeting.cpp

Requirements
  • You must include the standard header content used in all previous labs.
  • All identifiers must follow good naming conventions.
  • Follow proper whitespace practices including indentation, spacing, and brace placement.
  • Your program must compile and execute without warnings or errors.
Function Specifications

Your program must include:

  • A function prototype placed above main(). The prototype must include:
    • the return type
    • the function name greet
    • a clearly named parameter
    • a default value for the parameter
  • The full function definition must appear after the main() function.
  • The function must produce an appropriate greeting message using the supplied or default name.
Program Behavior
  • main() must call greet() at least twice.
  • One call must use the default parameter value.
  • Another call must provide a name argument explicitly.
  • Output must be neat, readable, and clearly labeled.
File Requirements & Submission

Add your source file to your lab folder.

  • Greeting.cpp — containing all code (header section, main(), prototype, and function definition).

Ensure your file is formatted correctly and follows the coding conventions introduced throughout the course.
