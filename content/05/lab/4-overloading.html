Lab Instructions: Writing and Calling Overloaded Functions

Overview

In this assignment, you will write a simple C++ program that demonstrates function overloading. You will create multiple functions that share the same name but differ by their parameter lists.

The goal is to help you understand how C++ chooses which version of an overloaded function to call based on the number and types of arguments.

File Requirements
  • Your entire program must be contained in a single file named OverloadingDemo.cpp.
  • Include the standard header content required in all previous assignments.
  • Follow proper naming conventions and consistent indentation.
  • Ensure all your function prototypes appear above main().
  • All full function definitions must appear after main().
Function Requirements

Your program must define at least three overloaded versions of the same function name.

These versions must differ by their parameter lists. For example:

  • Different number of parameters
  • Different parameter types
  • A mix of both

Each prototype must include parameter names, not just types, to support clarity and readability.

Program Behavior
  • The main() function must call each overloaded version at least once.
  • Output must clearly indicate which version of the function was executed.
  • All output should be clearly labeled, easy to read, and meaningful.
  • The program must compile and run without warnings or errors.
What to Submit
  • Your completed OverloadingDemo.cpp file.
  • Ensure all header content, prototypes, function definitions, and comments are included and formatted correctly.
  • Indentation, spacing, and identifier naming must follow course style conventions.
