Lab Exercise: Tracing Code Execution

In this assignment, you will practice careful code transcription, program tracing, and understanding program flow in C++. You will copy the program exactly as printed in the textbook for Program 6-14, and then create a full line-by-line execution trace for two different test runs.

This assignment requires precision, attention to detail, and understanding control flow, not writing original code.

In this lab exercise, you will practice tracing the execution of C++ code to understand how it works. You are given a simple C++ program and asked to follow the flow of execution, keeping track of variable values and function calls.

The ability to read and understand C++ code, is an essential skill for debugging and writing your own programs.

Part 1 - Copy the Textbook Program

Copy the Textbook Program

  1. Locate Program 6-14 in your textbook.
  2. Carefully copy the program line-by-line into a new file named HealthClubMembership.cpp. Make sure to include:
    • Whitespace
    • Indentation
    • Braces
    • Capitalization
    • Spacing

Notes:

  • For this assignment only, you do NOT include the standard header block usually required for programs in this course.
  • The purpose is to match the textbook exactly so the line numbers correspond during your trace.
Part 2 - Create Trace File

Create a plain text file named:

HealthMembershipTrace.txt

Notice, this is a text file. Do not create a C++ source file, or RTF document, or Word document. It must be a plain text file with the .txt extension.

Part 3 - Create Execution Trace #1
    Use the sample test data from the textbook:
  • Menu selection: 1
  • Input for months: 3

Instructions for the trace file:

    For every program statement executed:
  • Record the line number (matching the textbook version of Program 6-14)
  • Describe what happens on that line
  • Continue until the program exits

This should resemble a step-through as if you were the compiler/interpreter walking the program from top to bottom, including:

  • entering switch branches
  • evaluating conditions
  • updating variables
  • output operations
  • any jumps in control flow such as loops or function calls

Part 4 - Create Execution Trace #2
    Use different test data:
  • Menu selection: 2
  • Input for months: 6

Follow the same instructions as in Part 3 to create a second execution trace for this new set of inputs.

Add this second trace to the same text file, clearly labeled: 

--- Trace for Test Case #2 (Menu 2, Months 6) ---

Repeat the same process:

  • Record line number
  • Describe what happens on that line
  • Continue until the program exits

Submission Instructions

Save both files to your lab folder:

  • HealthClubMembership.cpp - the copied program source code
  • HealthMembershipTrace.txt - the execution trace file with both test cases

Ensure that your trace file is clear, organized, and accurately reflects the flow of execution for both test cases.

Grading Considerations
