Reference Parameters

In C++, a reference parameter allows a function to work directly with the original variable that was passed in, instead of a copy. This means the function can change the caller’s variable.

With regular parameters (pass‑by‑value), the function receives a copy of the argument, so any changes remain only inside the function. Reference parameters fix this by using the & symbol in the parameter list.

Why use reference parameters?

  • To allow a function to modify a variable in the caller
  • To avoid making copies of large objects (more efficient)
  • To return multiple results through parameters

A reference parameter acts like an alias — a second name for the same variable.

Demo Code: Reference Parameters in Action

In this example, the function triple() receives a reference parameter. Because of that, the function can change the actual variable from main(). 


 1 | #include <iostream>
 2 | using namespace std;
 3 |
 4 | void triple(int &num);   // reference parameter
 5 |
 6 | int main() {
 7 |     int value = 10;
 8 |     cout << "Before: " << value << endl;
 9 |
10 |     triple(value);        // passes the original variable
11 |
12 |     cout << "After:  " << value << endl;
13 |     return 0;
14 | }
15 |
16 | void triple(int &num) {
17 |     num = num * 3;        // modifies the original variable
18 | }

Tracing what happens:

  • Line 7: value is created and set to 10
  • Line 10: triple(value) sends the actual variable
  • Line 17: num refers to value, so changing num changes value
  • Line 12: value now contains 30

Because the parameter in triple() is a reference, the function does not work with a copy — it works with the real variable.

When to Use Pass‑by‑Value vs. Pass‑by‑Reference

In C++, choosing between pass‑by‑value and pass‑by‑reference depends on what you want the function to do and how much data you are working with. Here is a simple guide for beginners:

Use Pass‑by‑Value when:
  • You do not want the function to change the original variable.
  • You only need a copy of the data to work with.
  • You are passing small and inexpensive types, such as: int, double, char, or bool.
  • Safety and clarity matter more than micro‑efficiency.

Example:

void showSquare(int x) {   // value is copied
        cout << x * x;
    }
Use Pass‑by‑Reference when:
  • You want the function to modify the caller’s variable.
  • You are passing a large object (string, vector, struct, class) and want to avoid copying it.
  • You need to return multiple values through parameters.

Example:

void increase(int &value) {   // original value is modified
        value++;
    }
A Helpful Rule of Thumb
  • If the function should not change the argument → use value.
  • If the function must change the argument → use reference.
  • If the data is large and copying is wasteful → use reference or const reference.

For beginners, thinking about “copy vs. original” is usually the clearest way to choose the correct parameter type.
