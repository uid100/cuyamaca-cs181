What is a C-string?

A C-string is a sequence of characters stored in a char array and ended by a special character called the null terminator '\0'. That “zero” character marks the end of the text. 

char name[6] = {'A','l','i','c','e','\0'}; // 5 letters + '\0'
/* same as */
char name2[6] = "Alice"; // string literal adds '\0' automatically
Key idea: A C-string is just a char array with a '\0' at the end. The array must be big enough for all characters plus the null terminator.

Declaring and initializing

Two common ways to create a C-string:

char word1[10] = "hello"; // OK: compile-time initialization
char word2[10];
/* ...assign this later... */

The first line is an initialization with a string literal (adds '\0' for you). The second line just reserves space; it doesn’t set a value yet.

Definitions:
  • Declaration: telling the compiler about the variable's type and name.
  • Assignment: setting, or changing a variable's value.
  • Initialization: both assigning and declaring a variable at the same time.

Why strcpy()?

After an array exists, you can't assign a new C-string to it using the usual assignment operator ( '=' ). Arrays don't support assignment like variables do. 

char msg[20];
msg = "hello";   // ERROR: cannot assign to an array

Instead, you must copy characters into the array. That's what strcpy does: it copies the bytes from a source string (including the '\0') into your destination array. 

#include <cstring>  // for strcpy

char msg[20];
strcpy(msg, "hello");   // copies 'h' 'e' 'l' 'l' 'o' '\0' into msg
Important: Make sure your destination array is big enough for the text and the null terminator. If the text is longer than the array, you'll overwrite memory (a bug). (BAD!)

Variants you might see: strncpy (copy up to N chars; be careful—it may not add '\0'), or platform-specific strcpy_s (takes the destination size). For this class, the main idea is: use a copy function, not =, to change a C-string.

Reading input into a C-string

You can read directly into a char array with cin or cin.getline:

char name[30];

std::cin >> name;        // reads a word (stops at whitespace)
std::cin.ignore(1000, '\n'); // (optional) clear remaining input

std::cin.getline(name, 30);  // reads a whole line up to 29 chars + '\0'
Tip: Prefer cin.getline when you want spaces (e.g., full names). Always pass the array size so it won’t overflow.

Common mistakes (and fixes)

  • Forgetting space for '\0': char s[5] = "hello"; ERROR needs 6. Use char s[6] or let the compiler size it.
  • Using = after declaration: arrays can't be assigned; use strcpy.
  • Buffer too small: copying a longer string than the array size corrupts memory.
// Safer sizing at initialization:
char s[] = "hello"; // compiler picks size 6 automatically

// Later update:
char t[20];
strcpy(t, "short text"); // OK; fits (including '\0')
Safer:

c-strings are important but std::string is safer and easier.
