C++ Operators (PEMDAS), Precedence & Associativity

Operators

An operator is a symbol that tells C++ to do something with values.

  • + add, - subtract, * multiply, / divide
  • = assign (store a value), == compare two values
  • && logical AND, || logical OR, ! logical NOT
int total = 5 + 3;     // add
bool same = (5 == 3);  // compare (false)

Precedence (think PEMDAS)

Precedence is the “order of operations.” Use the math rule you already know: PEMDAS.

  • Parentheses
  • Exponents — not a built-in operator in C++ (use std::pow when allowed)
  • Multiplication & Division
  • Addition & Subtraction
int result1 = 2 + 3 * 4;   // 14: do multiplication first, then addition
int result2 = (2 + 3) * 4; // 20: parentheses change the order
Rule: When in doubt, add parentheses. Clear code > clever code.

Associativity (when priorities tie)

Associativity tells which direction we group when operators have the same precedence.

  • Left-to-right for most math: + - * / %
  • Right-to-left for assignment: =, +=, etc.
int v = 10 - 3 - 2;  // (10 - 3) - 2 = 5 (left-to-right)
int a, b, c;
a = b = c = 4;       // c=4, then b=c, then a=b (right-to-left)

Simple precedence checklist

  1. () parentheses
  2. * / % multiplication/division/remainder
  3. + - addition/subtraction
  4. < > <= >= comparisons
  5. == != equality
  6. && logical AND
  7. || logical OR
  8. = assignment (usually last)
bool r = 3 + 4 * 2 == 11 && true;
// Steps: (4*2)=8 → 3+8=11 → 11==11 true → true && true → true

Common mistakes (and quick fixes)

  • = vs ==: = assigns, == compares.
  • Integer division: 7/2 is 3, not 3.5. Use 7.0/2 or cast.
  • Clarity: When the order isn’t obvious, add parentheses.
// Wrong vs right
if (x = 5) { }   // assigns 5; condition becomes true (bug)
if (x == 5) { }  // compares correctly

int a = 7 / 2;       // 3
double b = 7.0 / 2;  // 3.5

Quick practice

  1. Predict the values, then test:
int a = 2 + 3 * 4;
int b = (2 + 3) * 4;
int c = 8 / 2 * 2;
int d = 5 - 2 - 1;
bool e = (3 * 3 == 9) || (4 < 2) && (10 > 1);
  1. Rewrite any unclear expression with parentheses to show the order.
