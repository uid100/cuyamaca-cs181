Assignment: Convert Sentence to camelCase

In programming, compound variable naming is used when a variable name contains multiple words combined into one identifier. Different naming conventions help improve readability and maintain consistency across codebases.

Common Naming Styles:

  • camelCase: First word lowercase, subsequent words capitalized
    myVariableName
  • PascalCase: Every word capitalized
    MyVariableName
  • snake_case: Words separated by underscores
    my_variable_name
  • kebab-case: Words separated by hyphens (not valid in C++ identifiers)
    my-variable-name

Conventions in C++:

  • Class names are typically written in PascalCase
  • Constants are often written in ALL_CAPS
  • Variables and functions commonly use camelCase

In this assignment, you will create a utility program that converts a sentence into camelCase.

Program Requirements
  • Prompt the user to enter a full sentence using getline().
  • Store the input using std::string.
  • Convert the sentence into camelCase format:
    • The first word should be entirely lowercase.
    • Each subsequent word should have its first letter capitalized.
    • All other letters should be lowercase.
    • All spaces should be removed in the final result.
  • Assume words are separated by one or more(!) spaces.
  • Output the resulting camelCase string.
Functional Expectations
  • Use std::string operations to build the result string.
  • Use loops to process each character in the input.
  • Use conditional logic to detect word boundaries.
  • Convert characters using functions like tolower() and toupper().
  • Do NOT use advanced libraries or functions not covered in class.
Example Program Behavior
Enter a sentence: hello world
Output: helloWorld

Enter a sentence: Programming is FUN
Output: programmingIsFun

Enter a sentence: convert THIS sentence now
Output: convertThisSentenceNow
Coding Standards & Requirements
  • Include the course-standardized header comment block.
  • Use appropriate headers (<iostream>, <string>, <cctype>).
  • Use meaningful variable names (e.g., input, result, newWord).
  • Follow consistent indentation and formatting.
  • Comment key parts of your logic clearly.
  • Test your program with different sentence formats.
Hints (Optional Guidance)
  • Track whether you are at the beginning of a new word.
  • Use a boolean flag like newWord = true.
  • Capitalize the first letter of each new word except the first.
  • Skip (all) spaces when building the result string.
  • There may be multiple spaces between words.
  • There may be leading spaces too. Don't assume the first character is not a space.
