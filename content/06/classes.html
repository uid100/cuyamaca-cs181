Understanding Abstraction

Abstraction is the process of hiding complex implementation details and showing only the necessary features of an object. It allows programmers to focus on what an object does rather than how it does it.

Real-World Examples:
  • A Microwave: You know how to set the timer and press "Start" (the interface), but you don't need to understand how the magnetron generates radio waves to heat your food.
  • A Car: You use the steering wheel and pedals to drive without needing to know the thermodynamics of the internal combustion engine.
Abstract Data Types (ADTs)

An Abstract Data Type (ADT) is a data type whose behavior is defined by a set of values and a set of operations. It is a theoretical model that specifies:

  1. The data being stored.
  2. The operations that can be performed on that data.

Note: An ADT tells you what operations are available, but not how they are coded. In C++, we use Classes to implement ADTs.

Classes and Objects

In C++, a Class is a blueprint or template for creating objects. An Object is a specific instance of that class.

Attributes (Data)

Variables that hold the state of the object (e.g., color, speed).

Methods (Functions)

Functions that define the behavior of the object (e.g., accelerate(), brake()).

Access Specifiers: Public vs. Private

Access specifiers define how the members (attributes and methods) of a class can be accessed.

  • Private: Members can only be accessed by code inside the class. This is used for "Data Hiding."
  • Public: Members can be accessed from outside the class using the object.
class BankAccount {
        private:
            double balance; // Cannot be accessed directly from main()

        public:
            void deposit(double amount) { // Can be called from main()
            balance += amount;
        }
    };
Accessors and Mutators

Since we typically keep data private, we use special public functions to interact with it:

Accessors (Getters): Functions that "get" or retrieve a value but do not change it. Usually marked as const.
Mutators (Setters): Functions that "set" or modify the value of a private attribute, often including validation logic.
Constructors

A Constructor is a special member function that is automatically called when an object is created. It is primarily used to initialize the object's attributes.

  • It has the same name as the class.
  • It has no return type (not even void).
class Rectangle {
                public:
                    // Constructor
                    Rectangle() {
                        width = 1.0;
                        height = 1.0;
                    }
                private:
                    double width, height;
            };
