A Pet Class

In this lab, you will practice creating a simple custom C++ class using separate header and implementation files, and then write a small driver program to instantiate and use several objects.

You will create a Pet class that stores a pet’s name, breed, age, and weight, along with a member function that prints a short description.

Lab Folder Setup

Updated: Optionally, you may create a subfolder named pets within your lab folder to keep these files organized. This is not required, but it may help keep your folder better organized and make it easier to compile the files together.

Add the following three files to your lab folder:

  • Pet.h — class specification (header)
  • Pet.cpp — class implementation
  • PetsDemo.cpp — application (driver)

Important: Each file must begin with the mandatory standard file header content required by this course.

Step 1: Create the Class Specification (Pet.h)

In Pet.h, define the Pet class interface. This file should describe what the class contains, but not how it works internally.

Your header file should include:

  • Include guards - Prevent multiple inclusions of the header file
  • The Pet class declaration
  • Private data members for:
    • name
    • breed
    • age
    • weight
  • Public member functions:
    • A constructor that receives all values as parameters
    • A function that prints a short description of the pet

Do not hard‑code any pet data in this file.

Step 2: Implement the Class (Pet.cpp)

In Pet.cpp, write the function definitions for the Pet class.

This file should:

  • Include "Pet.h"
  • Implement the constructor to initialize all data members
  • Implement the description function to display the pet’s information in a readable format

Again, do not hard-code specific pet values. The implementation must work with any data passed in from the program.

Step 3: Write the Driver Program (PetsDemo.cpp)

The PetsDemo.cpp file is the client of your class. This is the only place where sample data should appear.

In this file:

  • Include <iostream> and "Pet.h"
  • Create at least three Pet objects using sample data
  • Call the description function for each object

Example (conceptually): two dogs and one cat with different attributes.

Step 4: Build and Run

Compile all three files together: 


g++ PetsDemo.cpp Pet.cpp -o PetsDemo

Run the program and confirm that each pet's information is displayed correctly.

Submission Checklist

  • All three files are in the lab folder
  • Mandatory standard file header content is present in each file
  • No data is hard-coded in Pet.h or Pet.cpp
  • Multiple objects are created in PetsDemo.cpp
  • The program compiles and runs without errors
