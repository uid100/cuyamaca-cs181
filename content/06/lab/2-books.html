Building a Book Class and BookStore App

In this lab, you will create a simple Book class using proper C++ file separation and then write a small bookstore application that manages and displays an inventory of books.

This exercise reinforces:

  • Writing custom classes
  • Using private data with public accessors
  • Separating specification, implementation, and client code

Lab Folder Setup

Updated: Again, And still optional, you may create a subfolder named books within your lab folder to keep these files organized. This is not required, but it may help keep your folder better organized and make it easier to compile the files together.

Add the following three files to your lab folder:

  • Book.h — class specification (header)
  • Book.cpp — class implementation
  • BookStore.cpp — BookStore application (driver)

Each file must begin with the mandatory standard file header content required for this course.

Step 1: Create the Class Specification (Book.h)

In Book.h, declare the Book class. This file defines the interface only.

Your header file must include:

  • Include guards
  • Private data members:
    • title
    • author
    • number of pages
    • price
  • Public members:
    • A constructor that initializes all attributes
    • Getter functions for each attribute
    • A setter function for updating the sales price
    • A function to print a short description

Do not hard‑code any book data in this file.

Step 2: Implement the Class (Book.cpp)

In Book.cpp, implement all functions declared in the header.

This file should:

  • Include "Book.h"
  • Define the constructor and initialize all private attributes
  • Define each getter to return the correct value
  • Define the setter to update the book’s price
  • Format the description output in a readable way

The class must work with any data passed in by the application.

Step 3: Build the BookStore Application (BookStore.cpp)

The BookStore.cpp file represents a simple bookstore program and acts as the client of the Book class.

In this file:

  • Include <iostream>, and "Book.h"
  • Create a small inventory of Book objects). Optionally, you can use a collection such as vector or array.
  • Include sample book data here only
  • Write a function named something like printInventory() that prints all books in the inventory
  • Call the inventory function from main()

Step 4: Compile and Run


g++ BookStore.cpp Book.cpp -o bookstore

Run the program and verify that the inventory displays correctly and that each book’s description is shown.

Submission Checklist

  • All three files are included in the lab folder
  • Mandatory standard file header content is present in each file
  • No data is hard‑coded in Book.h or Book.cpp
  • Private data is accessed only through public functions
  • The program compiles and runs without errors
