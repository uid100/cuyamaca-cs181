Structs: Introduction

Introducing Structs in C++

As C++ programs grow, we often need to group related pieces of data together. One of the simplest tools for doing this is the struct. Structs allow you to combine multiple variables into a single, meaningful unit.

This page introduces structs, shows how to use them with examples, and explains how they compare to classes.

What Is a Struct?

A struct (short for structure) is a user-defined data type that groups variables of different types under a single name.

Each variable inside a struct is called a member. Structs are commonly used to represent objects that are mainly collections of data.

Basic Struct Example


struct Student
{
    int id;
    std::string name;
    double gpa;
};

This struct defines a new type called Student that groups together an ID number, a name, and a GPA.

Creating and Using Struct Variables


#include <iostream>
#include <string>

int main()
{
    Student s1;
    s1.id = 12345;
    s1.name = "Alex Kim";
    s1.gpa = 3.75;

    std::cout << s1.name << " has GPA " << s1.gpa << std::endl;
    return 0;
}

You access struct members using the dot (.) operator. Each variable behaves like a normal variable, but is grouped inside the struct.

Structs and Functions

Structs can be passed to functions, just like any other data type. 


void printStudent(const Student& s)
{
    std::cout << s.id << " - " << s.name << std::endl;
}

Passing by reference avoids copying and keeps the function efficient.

Structs vs. Classes

But which one to use?

Structs and classes in C++ are very similar. In fact, they support the same features, including constructors, methods, and access control. The key differences are default access and typical usage.

  • Structs
    • Members are public by default
    • Primarily used to group data
    • Often have little or no behavior
  • Classes
    • Members are private by default
    • Used to model objects with behavior
    • Encapsulate data and enforce rules

Structs Can Have Functions Too

Even though structs are often simple, they can include member functions. 


struct Point
{
    double x;
    double y;

    double distanceFromOrigin() const
    {
        return std::sqrt(x * x + y * y);
    }
};

This looks very similar to a class—because underneath, it essentially is.

When to Use a Struct

  • When the type mainly stores data
  • When all members should be public
  • When modeling simple records or values
  • When learning or prototyping simple designs

Examples include coordinates, configuration data, or simple records read from a file.

When to Use a Class

  • When data must be protected or controlled
  • When enforcing rules or invariants
  • When modeling real-world objects with behavior
  • When designing large or reusable systems

As your programs grow more complex, classes become the more appropriate choice.

Key Takeaway

Structs are a powerful and simple way to bundle related data together. While structs and classes are technically very similar in C++, they serve different design purposes. Understanding when to use each is an important step toward writing clean, professional C++ code.
