What are Nested Loops?

Nested loops are loops placed inside the body of another loop. The outer loop runs multiple times, and for each iteration of the outer loop, the inner loop runs completely.

  • Common when working with grids, tables, matrices, or patterns.
  • Any loop type can be nested: for inside for, while inside for, etc.
// General idea (pseudocode)
outer_loop {
    // do something before inner loop
    inner_loop {
        // do something repeatedly for each outer iteration
    }
    // do something after inner loop
}
Common Syntax Patterns (C++)
// for-in-for: printing a rows x cols grid of '*'
#include <iostream>
using namespace std;

int main() {
    int rows = 3, cols = 5;
    for (int r = 0; r < rows; ++r) {            // outer loop controls rows
        for (int c = 0; c < cols; ++c) {        // inner loop controls columns
            cout << "*";
        }
        cout << "\n";                         // end of a row
    }
}

 

// while-in-for: same idea, different structure
for (int r = 0; r < rows; ++r) {
    int c = 0;
    while (c < cols) {
        cout << "*";
        ++c;
    }
    cout << "\n";
}

 

// do-while inside for: guarantees the inner loop runs at least once
for (int r = 0; r < rows; ++r) {
    int c = 0;
    do {
        cout << "*";
        ++c;
    } while (c < cols);
    cout << "\n";
}
Tracing Execution (Mentally Step Through)
int rows = 2, cols = 3;
for (int r = 0; r < rows; ++r) {
    for (int c = 0; c < cols; ++c) {
        cout << "(" << r << "," << c << ") ";
    }
    cout << "\n";
}
// Output:
// (0,0) (0,1) (0,2)
// (1,0) (1,1) (1,2)

 

  1. r=0: inner loop prints (0,0) (0,1) (0,2).
  2. r=1: inner loop prints (1,0) (1,1) (1,2).

// Times table (1..n)
int n = 5;
for (int i = 1; i <= n; ++i) {
    for (int j = 1; j <= n; ++j) {
        cout << (i * j) << "\t";
    }
    cout << "\n";
}
Common Mistakes
  • Off-by-one errors : using <= instead of < (or vice versa) and producing an extra iteration.
  • Resetting inner counters : ensure inner loop variables are initialized inside the outer loop.
  • Infinite inner loops : always update the inner loop condition variables.
  • Misusing break/continue : remember break exits only the nearest loop.
  • Variable scope : loop indices declared in for (...) exist only within that loop’s scope.
Performance & Big-O Intuition

Two loops, each running n times, often lead to roughly n × n = n² operations. Three nested loops of size n lead to ~n³. This matters for large inputs.

// Example: O(n^2)
int n = 1000; long long sum = 0;
for (int i = 0; i < n; ++i) {
    for (int j = 0; j < n; ++j) {
        sum += i + j; // ~ n^2 operations
    }
}
When to Use vs. Avoid
  • Use: tables, 2D arrays/matrices, character graphics, comparing all pairs, building patterns.
  • Avoid (or refactor): when there’s an available formula, library algorithm, or data structure that reduces complexity.
// All pairs sum (naive): O(n^2)
long long sumPairs(const vector& a) {
    long long s = 0;
    for (size_t i = 0; i < a.size(); ++i)
        for (size_t j = i + 1; j < a.size(); ++j)
            s += a[i] + a[j];
    return s;
}

// Hollow rectangle challenge
int rows = 5, cols = 10;
for (int r = 0; r < rows; ++r) {
    for (int c = 0; c < cols; ++c) {
        bool border = (r == 0) || (r == rows - 1) || (c == 0) || (c == cols - 1);
        cout << (border ? '#' : ' ');
    }
    cout << "\n";
}
Mini-Projects / Applications
  • Multiplication table : format with tabs or fixed widths.
  • Image processing concept : iterate over rows/cols of a pixel grid (no libraries needed to discuss).
  • Game board : print a tic-tac-toe or minesweeper board.
  • String combinations: generate all pairs of characters from two strings.
// All pairs of characters from two strings
#include <iostream>
#include <string>
using namespace std;

int main() {
    string A = "ab", B = "XYZ";
    for (char a : A) {
        for (char b : B) {
            cout << a << b << ' ';
        }
    }
    // Output: aX aY aZ bX bY bZ
}
Debugging & Tracing Tips
  • Add cout statements that print the indices before/after the inner loop.
  • Start with tiny sizes (e.g., 2×3) so you can fully trace output.
  • Check boundary conditions: what happens for 0, 1, and typical values?
  • Prefer size_t for indexing vectors/arrays; avoid mixing signed and unsigned without care.
for (int r = 0; r < rows; ++r) {
    cout << "r=" << r << " start\n";
    for (int c = 0; c < cols; ++c) {
        cout << "  c=" << c << "\n";
    }
}

// Count evens in a 2D vector
int countEven(const vector>& grid) {
    int cnt = 0;
    for (size_t r = 0; r < grid.size(); ++r) {
        for (size_t c = 0; c < grid[r].size(); ++c) {
            if (grid[r][c] % 2 == 0) ++cnt;
        }
    }
    return cnt;
}
