Incrementing & Decrementing in C++

Updating a variable by one step is a common operation in programming. C++ provides special operators to make this concise and efficient.

We call this incrementing (adding 1) and decrementing(subtracting 1).

Basic Increment & Decrement

C++ provides special operators for adding or subtracting 1 from a variable:

  • x++      increment by 1
  • x--      decrement by 1
  • ++x      also increments by 1
  • --x      also decrements by 1

Examples

int x = 5;
x++;     // now x = 6
x--;     // now x = 5 again

Prefix vs Postfix

Both ++x and x++ add 1 to x, but they differ in when the increment happens.

Postfix (x++)

Return the old value, then increment.

int a = 5;
int b = a++;   // b = 5, then a becomes 6

Prefix (++x)

Increment first, then return the new value.

int a = 5;
int b = ++a;   // a becomes 6, then b = 6

When Should You Use Each?
  • Prefix (++x) — often preferred inside loops, especially with objects.
  • Postfix (x++) — useful when you explicitly need the old value first.

Compound Assignment Operators

More shorthand operators exist for combining arithmetic operations with assignment. These are called compound assignment operators.

Compound assignment updates a variable in place, combining an operation with assignment. These are shorthand for common patterns:

 

// Arithmetic
    x += 5;   // x = x + 5
    x -= 3;   // x = x - 3
    x *= 2;   // x = x * 2
    x /= 4;   // x = x / 4
    x %= 10;  // x = x % 10

 

// Bitwise
m &= mask;  // m = m & mask
m |= mask;  // m = m | mask
m ^= mask;  // m = m ^ mask
m <<= 1;    // m = m << 1
m >>= 2;    // m = m >> 2

Why Use Them?

  • Clarity: Express intent concisely ("add and store").
  • Maintainability: Less repetition of the variable name reduces mistakes.
  • Performance: For user-defined types/iterators, the shorthand may avoid extra temporaries (implementation-dependent).

Gotchas & Best Practices
  • Watch out for integer division with /= — results truncate toward zero for integers.
  • %= requires an integer (or integral type); avoid with floating‑point.
  • Bitwise compound ops operate on bits, not booleans or logical conditions.
  • Avoid writing expressions with multiple side effects in one line if it hurts readability.
