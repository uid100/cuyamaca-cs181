Getting Started
hello.cpp - step 1

Create a new C++ source file named hello.cpp and add the following code: 

    #include <iostream>

    int main() {
        std::cout << "Hello!" << std::endl;
        return 0;
    }

This simple C++ program consists of the following components:

  • Preprocessor Directives: The line #include <iostream> is a preprocessor directive that tells the compiler to include the standard input-output stream library, which is necessary for using std::cout and std::endl.
  • Main Function: The int main() function is the entry point of the program. Execution starts from this function.
  • Output Statement: The line std::cout << "Hello!" << std::endl; outputs the text "Hello!" to the console, followed by a newline.
  • Return Statement: The line return 0; indicates that the program has completed successfully.

Save your file and compile it from the command line. Use the following command: 

g++ hello.cpp

If there are any warnings or errors, review your code for typos.

If the compilation is successful, an executable file, typically named a.out on Unix-like systems or hello.exe on Windows, will be created. Run the executable with the following command and observe the output: 

 % ./a.out 

 Hello!
hello.cpp - step 2

From the textbook chapter, we saw that the typical pattern for applications includes collecting input, processing the input, and producing output. Our simple program so far only produces output. The C++ standard library provides a simple way to write to the console using the std::cout object. Add to the program to gather input from the user using the std::cin object. 

    #include 
    // #include 

    int main()
    {
        std::string userName;

        std::cout << "Hello!" << std::endl;
        std::cout << "What's your name? ";
        std::cin >> userName;
        std::cout << "Hello, " << userName << '.' << std::endl;
        std::cout << "\tIs it ok if I call you " << userName << '?' << std::endl;

        return 0;
    }
hello.cpp - step 3

Better.

This program works, but there are a couple of issues. First, the program uses the std::string type to hold the user's name, but C++ doesn't really know what a string is unless we include the appropriate header file. Some compilers may allow this code to compile without the header, and it almost accidentally works because the string header is indirectly included by the iostream header. However, this is not guaranteed behavior and relying on it can lead to portability issues. To fix this, we should explicitly include the <string> header at the beginning of our program.

Also, we are repeatedly using the std:: prefix before standard library components like cout, cin, and string throughout the program. This can make the code verbose and harder to read. To simplify this, we can use the using namespace std; directive after our include statements. This allows us to use standard library components without the std:: prefix. 

    #include 
    #include 

    using namespace std;

    int main()
    {
        string userName;

        cout << "Hello!" << endl;
        cout << "What's your name? ";
        cin >> userName;
        cout << "Hello, " << userName << '.' << endl;
        cout << "\tIs it ok if I call you " << userName << '?' << endl;

        return 0;
    }
