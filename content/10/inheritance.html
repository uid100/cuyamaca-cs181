Inheritance

Class Inheritance

The textbook alludes to general and specialized versions of classes.

Inheritance is a fundamental object-oriented programming concept that allows a new class (called a derived class) to inherit properties and behaviors (data members and member functions) from an existing class (called a base class). This promotes code reusability and establishes a natural hierarchical relationship between classes.

A note on terminology.

You will hear and see the terms:

  • Base class and derived class
  • Parent class and child class
  • Super class and sub class
  • General class and specialized class
These terms are often used interchangeably to describe the relationship between the class that is being inherited from (base/parent/super/general) and the class that is inheriting (derived/child/sub/specialized).

Each of these pairs of terms can be used to describe the same relationship between classes. For now, the choice of terminology is not important, and you can use any of these terms to describe the inheritance relationship.

Inheritance in C++

In C++, inheritance is implemented using the ':' symbol. The syntax for declaring a derived class is as follows: 

                class DerivedClass : accessSpecifier BaseClass {
                    // Additional members and functions for the derived class
                };

The 'accessSpecifier' can be 'public', 'protected', or 'private', and it determines the accessibility of the inherited members in the derived class. The most common access specifier used for inheritance is 'public', which allows the derived class to access the public members of the base class.

For example, if we have a base class called 'Animal' and a derived class called 'Dog', we can declare the derived class as follows: 

                class Dog : public Animal {
                    // Additional members and functions for the Dog class
                };

In this example, the 'Dog' class inherits from the 'Animal' class using public inheritance. This means that the 'Dog' class can access the public members of the 'Animal' class, and it can also add its own members and functions to extend the functionality of the 'Animal' class.

Inheritance allows us to create a new class that is a specialized version of an existing class, which promotes code reusability and helps to establish a natural hierarchical relationship between classes. The derived class can override the inherited members to provide specific implementations or add new members to enhance the functionality of the base class.
