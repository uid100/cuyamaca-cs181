Static Class Members

Static Members

Static members are shared across all instances of a class. They belong to the class itself rather than any specific instance. Static members can be accessed without creating an instance of the class.

The textbook says:

"If a member function is declared static, it may be called before any instances of the class are defined."

In C++, static members are declared using the 'static' keyword. They can include static variables, static methods, and static properties.

Static members are often used for utility functions, constants, or to maintain state that is shared across all instances of a class.

Static Variables

Static variables are shared among all instances of a class. They are initialized only once and retain their value across all instances.

For example, a static variable can be used to count the number of instances of a class that have been created. 

                class Customer {
                public:
                    static int customerCount; // Static variable declaration
                    Customer() {
                        customerCount++; // Increment the static variable in the constructor
                        int id = customerNumber; // Access the static variable
                        customerNumber = customerCount; // Assign the static variable to the instance variable
                    }
                };

In this example, the static variable 'customerCount' is shared among all instances of 'MyClass'. Each time a new instance is created, 'customerCount' is incremented, allowing us to keep track of how many instances have been created.

Each new Customer that is created will have its own unique customer number, but they will all share the same customerCount variable that keeps track of the total number of customers.

Static Methods

Static methods are functions that belong to the class rather than any instance. They can be called without creating an instance of the class.

Static methods can only access static members of the class. They cannot access instance members because they do not have a reference to any specific instance.

Static methods are often used for utility functions that do not require access to instance data.

For example, a collection of related utility functions can be implemented as static methods within a class, where the instance of a class is not necessary to use those functions. 

                class MathUtils {
                public:
                    static int add(int a, int b) {
                        return a + b; // Static method that adds two numbers
                    }
                };
